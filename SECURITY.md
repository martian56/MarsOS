# Security Policy

## Supported Versions

Security fixes are provided for the latest development line.

| Version | Supported |
| --- | --- |
| `main` | yes |
| Latest tagged release | best effort |
| Older tags | no |

## Reporting a Vulnerability

Please do not report vulnerabilities in public issues.

Use GitHub private advisories:

- https://github.com/martian56/MarsOS/security/advisories/new

When reporting, include:

- affected commit/tag
- impact and attack surface
- proof of concept or reproduction steps
- suggested mitigation if available

## Response Targets

- Initial triage acknowledgement: within 5 business days
- Status update after triage: within 10 business days

These targets are goals, not guarantees.

## Disclosure Process

1. We validate and triage the report.
2. We develop and test a fix.
3. We publish a release and advisory details once users can patch safely.
